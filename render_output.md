# Render Output Structure

After executing the `RenderEngine` compilation pipeline, assets are systematically stored across the following structural hierarchy within the configured output directory:

```
output/
├── generated/
│   └── main.py
└── media/
    ├── images/
    │   └── main/
    └── videos/
        └── main/
            └── 720p30/
                └── AxiomGeneratedScene.mp4

```

## Directory Reference Matrix

| Directory | Core Purpose |
| --- | --- |
| `generated/` | Holds the raw, compiled Python source code strings (`main.py`) generated by `ManimGenerator` before framework ingestion. |
| `media/` | The global distribution root designated for all raw artifact binary exports produced by the Manim engine. |
| `images/` | Stores static vector snapshots, design rasterized checkframes, and presentation-ready graphic frames extracted from runtime layers. |
| `videos/` | Contains the complete, final rendered `.mp4` video assets structured cleanly by relative canvas resolution and target frame rates. |

> **Note:** The layout abstracts development sources from final binary distributions, maintaining stateless determinism during consecutive rendering cycles.
